How This Website Is Built

A Comprehensive Guide for Beginners

Introduction

Welcome! This guide will teach you how the Quantum Investor Digest website is built from scratch. By the end of this tutorial, you'll understand every component and be able to create a similar website yourself. We'll cover the HTML structure, CSS styling, JavaScript functionality, and the overall architecture.

What you'll learn: HTML document structure, CSS styling and responsive design, JavaScript for dynamic behavior, file organization, and best practices for modern web development.

1. Website Architecture & File Structure

Understanding the Framework

This is a static website, meaning the pages are pre-built HTML files that don't require a server-side language like PHP or a database. This makes it fast, secure, and easy to host.

File Organization

My-blog/
├── index.html           (Homepage)
├── about.html           (About page)
├── education.html       (This page)
├── Disclosures.html     (Legal disclosures)
├── styles.css           (All styling)
├── CNAME                (Domain configuration)
│
├── js/                  (JavaScript files)
│   ├── template-loader.js   (Loads header/footer)
│   ├── mobile-menu.js       (Mobile navigation)
│   └── tldr.js              (Dashboard metrics)
│
├── templates/           (Reusable components)
│   ├── header.html          (Site header)
│   └── footer.html          (Site footer)
│
├── Media/               (Images & graphics)
│   └── PostImages/
│
├── Posts/               (Blog articles)
│   └── posts.html
│
└── Data/                (JSON data files)
    └── master.json          (Portfolio data)

Why this structure? Separation of concerns - HTML for content, CSS for styling, JavaScript for behavior, and data in JSON files. This makes the site maintainable and scalable.

2. HTML: The Structure

What is HTML?

HTML (HyperText Markup Language) is the skeleton of every webpage. It defines the structure and content using tags (elements enclosed in angle brackets like <div>).

Anatomy of an HTML Document

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
</body>
</html>

Breaking It Down:

  • <!DOCTYPE html> - Tells the browser this is HTML5
  • <html lang="en"> - Root element, language is English
  • <head> - Contains metadata (not visible on page)
  • <meta charset="UTF-8"> - Character encoding for text
  • <meta name="viewport"...> - Makes site responsive on mobile
  • <title> - Shows in browser tab
  • <link rel="stylesheet"> - Loads CSS file
  • <body> - Contains visible content

Key HTML Elements Used

Semantic HTML5 Tags

  • <header> - Site header with logo and navigation
  • <main> - Main content area
  • <article> - Self-contained content (blog posts)
  • <section> - Thematic grouping of content
  • <nav> - Navigation links
  • <footer> - Site footer
  • <h1>, <h2>, <h3> - Headings (h1 is most important)
  • <p> - Paragraphs
  • <a href=""> - Links
  • <img src="" alt=""> - Images
  • <div> - Generic container

SEO Meta Tags

These tags in the <head> help search engines understand your page: 

<meta name="description" content="Page description for Google">
<meta name="author" content="Michael Gavrilov">
<link rel="canonical" href="https://quantuminvestor.net/page.html">

<!-- Open Graph (for social media sharing) -->
<meta property="og:title" content="Page Title">
<meta property="og:description" content="Description">
<meta property="og:image" content="image-url.jpg">

<!-- Twitter Cards -->
<meta name="twitter:card" content="summary_large_image">

3. CSS: The Styling

What is CSS?

CSS (Cascading Style Sheets) controls how HTML elements look - colors, sizes, spacing, layout, and animations. This site uses one main CSS file (styles.css) that contains all styling rules.

CSS Syntax

selector {
    property: value;
}

/* Example */
h1 {
    font-size: 2.25rem;
    color: #ffffff;
    margin-bottom: 1.5rem;
}

CSS Variables (Custom Properties)

This site uses CSS variables to store colors and values that can be reused throughout the stylesheet: 

:root {
    --primary: #a855f7;           /* Purple accent color */
    --primary-dark: #7c3aed;      /* Darker purple */
    --bg-black: #000000;          /* Background black */
    --bg-gray-900: #111111;       /* Dark gray */
    --bg-gray-800: #1a1a1a;       /* Medium gray */
    --text-white: #E0E0E0;        /* Off-white text */
    --text-gray-400: #9ca3af;     /* Gray text */
    --border-gray-800: #1a1a1a;   /* Border color */
}

/* Use variables like this: */
.button {
    background-color: var(--primary);
    color: var(--text-white);
}

Why use variables? If you want to change the purple color site-wide, you only need to update --primary in one place instead of hundreds of locations.

Utility-First CSS Classes

This site uses a utility-first approach (inspired by Tailwind CSS). Small, reusable classes that do one thing: 

/* Spacing */
.px-4 { padding-left: 1rem; padding-right: 1rem; }
.py-12 { padding-top: 3rem; padding-bottom: 3rem; }
.mb-8 { margin-bottom: 2rem; }

/* Layout */
.flex { display: flex; }
.grid { display: grid; }
.items-center { align-items: center; }
.justify-between { justify-content: space-between; }

/* Typography */
.text-xl { font-size: 1.25rem; }
.font-bold { font-weight: 700; }
.text-gray-400 { color: var(--text-gray-400); }

/* Example usage in HTML */
<div class="flex items-center justify-between px-4 py-12">
    <h1 class="text-xl font-bold">Title</h1>
</div>

Responsive Design

The site adapts to different screen sizes using media queries: 

/* Desktop (default) */
.grid {
    grid-template-columns: repeat(3, 1fr);
}

/* Tablet (1024px and below) */
@media (max-width: 1024px) {
    .grid {
        grid-template-columns: repeat(2, 1fr);
    }
}

/* Mobile (640px and below) */
@media (max-width: 640px) {
    .grid {
        grid-template-columns: 1fr;
    }
}

Breakpoints used:
• Desktop: Default styles
• Tablet: max-width: 1024px and 768px
• Mobile: max-width: 640px and 375px

CSS Grid & Flexbox

Two powerful layout systems used throughout the site:

Grid (for 2D layouts)

.grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);  /* 3 equal columns */
    gap: 1.5rem;                            /* Space between items */
}

Flexbox (for 1D layouts)

.flex {
    display: flex;
    align-items: center;       /* Vertical centering */
    justify-content: space-between;  /* Space items apart */
    gap: 1rem;                 /* Space between items */
}

4. JavaScript: The Behavior

What is JavaScript?

JavaScript makes websites interactive and dynamic. It can respond to user actions, load content, manipulate the page, and fetch data from external sources.

Template Loader (template-loader.js)

This script loads the header and footer on every page, so you don't have to copy/paste them: 

// 1. Finds elements with data-template attribute
<div data-template="header" data-root-path=""></div>

// 2. Fetches the template file
fetch('templates/header.html')

// 3. Replaces placeholder with template content
element.innerHTML = html;

// 4. Handles relative paths
// Root pages use: data-root-path=""
// Posts folder uses: data-root-path="../"

Mobile Menu (mobile-menu.js)

Controls the hamburger menu on mobile devices: 

// 1. Get DOM elements
const menuButton = document.getElementById('mobile-menu-button');
const menuOverlay = document.getElementById('mobile-menu-overlay');

// 2. Open menu function
function openMenu() {
    menuOverlay.classList.add('active');  // Shows overlay
    document.body.style.overflow = 'hidden';  // Prevents scrolling
}

// 3. Close menu function
function closeMenu() {
    menuOverlay.classList.remove('active');  // Hides overlay
    document.body.style.overflow = '';  // Restores scrolling
}

// 4. Event listeners
menuButton.addEventListener('click', openMenu);
menuClose.addEventListener('click', closeMenu);

Key JavaScript Concepts

  • DOM (Document Object Model) - JavaScript representation of HTML
  • querySelector/getElementById - Find elements in HTML
  • addEventListener - Respond to user actions (clicks, keyboard)
  • fetch() - Load data from files or APIs
  • async/await - Handle asynchronous operations
  • classList - Add/remove CSS classes dynamically
  • IIFE - Immediately Invoked Function Expression (keeps code private)

Script Loading

<script src="js/template-loader.js" defer nonce="qi123"></script>

/* defer = Wait until HTML is parsed before executing
   nonce = Security feature (Content Security Policy)
   Ensures only authorized scripts run */

5. Component Architecture

Reusable Templates

The header and footer are stored in separate files and loaded dynamically:

templates/header.html

<header class="border-b border-gray-800 py-6">
    <div class="container mx-auto px-4">
        <div class="flex items-center justify-between">
            <a href="{{rootPath}}index.html">
                <img src="{{rootPath}}Media/Full-Logo.webp" alt="Logo">
            </a>
            <nav>
                <a href="{{rootPath}}index.html">Home</a>
                <a href="{{rootPath}}Posts/posts.html">Blog</a>
                <a href="{{rootPath}}about.html">About</a>
            </nav>
        </div>
    </div>
</header>

/* {{rootPath}} is replaced with "" for root pages
   or "../" for pages in subfolders */

Benefits of This Approach

  • DRY (Don't Repeat Yourself) - Update header once, changes everywhere
  • Maintainability - Easier to manage and update
  • Consistency - Same header/footer across all pages
  • Performance - Templates cached by browser after first load

What is JSON?

JSON (JavaScript Object Notation) is a format for storing and exchanging data. It's human-readable and easy for machines to parse.

Loading JSON with JavaScript

// Fetch data from JSON file
async function loadData() {
    const response = await fetch('Data/master.json');
    const data = await response.json();
    
    // Use the data
    console.log(data.portfolio_totals.current_value);
}

Why separate data from HTML? You can update portfolio numbers without touching HTML code. The TLDR strip and charts load data dynamically from this JSON file.

7. Security & Performance Best Practices

Content Security Policy (CSP)

Protects against XSS (Cross-Site Scripting) attacks while allowing common blog features: 

<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; 
               script-src 'self' 'nonce-qi123' 'unsafe-inline' 
                          https://www.googletagmanager.com 
                          https://www.google-analytics.com 
                          https://cdn.jsdelivr.net https://cdnjs.cloudflare.com; 
               style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; 
               img-src 'self' data: https: http:; 
               font-src 'self' data: https://fonts.gstatic.com; 
               connect-src 'self' https://www.google-analytics.com; 
               frame-src 'self' https://www.youtube.com;">

/* Blog-Friendly CSP:
   - Allows inline styles for flexibility
   - Permits popular CDNs (jsdelivr, cdnjs)
   - Enables Google Analytics tracking
   - Supports Google Fonts
   - Allows YouTube/Vimeo embeds
   - Still blocks unauthorized external scripts */

Image Optimization

<img src="Media/Hero.webp" 
     alt="Descriptive alt text"
     width="1200" 
     height="800"
     loading="lazy"           <!-- Lazy load off-screen images -->
     fetchpriority="high"     <!-- Priority for important images -->
     decoding="async">        <!-- Decode without blocking -->

/* WebP format = Smaller file size, faster loading
   Alt text = Accessibility for screen readers
   Width/height = Prevents layout shift */

Performance Optimizations

  • Minification - Remove unnecessary characters from code
  • Caching - Browser stores files locally for faster repeat visits
  • Preload - Load critical resources early
  • Defer scripts - Load JavaScript after HTML parsing
  • Responsive images - Serve appropriate sizes for device

8. Accessibility (a11y)

Why Accessibility Matters

Making websites usable by everyone, including people with disabilities who use screen readers or keyboard navigation.

Key Accessibility Features

/* Semantic HTML */
<main role="main">     <!-- Identifies main content -->
<nav aria-label="Primary">  <!-- Labels navigation type -->

/* Alt text for images */
<img src="logo.webp" alt="Quantum Investor Digest Logo">

/* Skip link for keyboard users */
<a href="#main-content" class="skip-link">Skip to main content</a>

/* ARIA attributes */
<button aria-label="Open mobile menu" aria-expanded="false">

/* Focus styles for keyboard navigation */
a:focus-visible {
    outline: 2px solid var(--primary);
    outline-offset: 2px;
}

10. Building Your Own Website

Step-by-Step Guide

  1. Set up your project folder
    • Create folders: css/, js/, images/, templates/
    • Create: index.html, styles.css
  2. Build the HTML structure
    • Start with index.html
    • Add DOCTYPE, html, head, body tags
    • Add meta tags for SEO and responsiveness
  3. Create the header and footer templates
    • Save as templates/header.html and footer.html
    • Use {{rootPath}} placeholders for paths
  4. Write your CSS
    • Define CSS variables for colors
    • Create utility classes
    • Add media queries for responsive design
  5. Add JavaScript functionality
    • Copy template-loader.js to load templates
    • Add mobile-menu.js for navigation
    • Test in browser console
  6. Test locally
    • Use VS Code Live Server extension
    • Or run: python -m http.server 8000
    • Check responsive design in DevTools
  7. Deploy to GitHub Pages
    • Push to GitHub repository
    • Enable Pages in repository settings
    • Visit your live site!

  • Browser - Chrome/Firefox with DevTools
  • Version Control - Git and GitHub
  • Image Optimization - Squoosh.app or TinyPNG
